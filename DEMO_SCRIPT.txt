[1:00 — ARCHITECTURE]
(show assets/architecture_diagram.png)
Let's walk through the architecture flow. (pause)
First, every query enters the system through our Security Checkpoint node, which scrubs PII and checks for prompt injections.
If cleared, the flow routes to the Orchestrator node, which uses our home_keeper_orchestrator agent.
The orchestrator delegates tasks dynamically using AgentTools to either the chore_manager_agent or the maintenance_scheduler_agent.
These sub-agents connect via stdio transport to our MCP Server to query and update the database.
If a high-priority action is requested, the orchestrator routes to the approval_node, which pauses execution and triggers a human confirmation interrupt before finishing.
